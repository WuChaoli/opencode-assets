# 成本估算指南

## 估算公式

```
成本 = (输入 tokens / 1,000,000) × 输入单价 + (输出 tokens / 1,000,000) × 输出单价
```

## 典型场景 Token 估算

| 场景 | 输入 tokens | 输出 tokens | 总计 |
|------|------------|------------|------|
| 简单查询 | 2K | 3K | 5K |
| 日常编码 | 30K | 20K | 50K |
| 代码审查 | 50K | 10K | 60K |
| 复杂重构 | 100K | 100K | 200K |
| 大型项目 | 300K | 200K | 500K |

## 各模型成本示例（以 50K tokens 为例）

| 模型 | 输入成本 | 输出成本 | 总计 |
|------|---------|---------|------|
| Qwen3.6 Free | $0 | $0 | $0 |
| Haiku 4.5 | $0.03 | $0.10 | $0.13 |
| Sonnet 4.6 | $0.09 | $0.30 | $0.39 |
| Opus 4.6 | $0.15 | $0.50 | $0.65 |
| GPT 5.3 Codex | $0.05 | $0.28 | $0.33 |

## 月度成本预估

假设每日使用量：

| 使用强度 | 日 tokens | 月 tokens | Sonnet 4.6 月成本 |
|---------|----------|----------|------------------|
| 轻度 | 100K | 3M | ~$12 |
| 中度 | 500K | 15M | ~$60 |
| 重度 | 1M | 30M | ~$120 |
| 团队 | 5M | 150M | ~$600 |

## 节省成本建议

1. **使用缓存**：缓存读取价格低（Sonnet $0.30/1M）
2. **选择合适模型**：简单任务用 Haiku
3. **精简 Prompt**：减少不必要的上下文
4. **利用免费模型**：预算有限时用免费模型
5. **设置月度限额**：Zen 平台可设置 spending limit
