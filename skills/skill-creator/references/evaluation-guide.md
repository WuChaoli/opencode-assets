# 评估标准与决策流程

## 评估维度

### 1. 功能匹配度（0-100%）

| 分数 | 标准 |
|------|------|
| 90-100% | 完全覆盖用户需求，几乎不需要修改 |
| 70-89% | 覆盖主要需求，需要少量修改 |
| 50-69% | 覆盖部分需求，需要较多修改 |
| 30-49% | 只有部分相关，需要大量修改 |
| 0-29% | 基本不相关 |

### 2. 质量指标

| 指标 | 优秀 | 良好 | 一般 | 较差 |
|------|------|------|------|------|
| Star 数 | > 100 | 20-100 | 5-20 | < 5 |
| 更新日期 | < 1 月 | 1-3 月 | 3-6 月 | > 6 月 |
| 维护状态 | 活跃维护 | 偶尔维护 | 很少维护 | 停止维护 |
| 文档质量 | 完整清晰 | 基本完整 | 不完整 | 缺失 |

### 3. 兼容性

| 检查项 | 标准 |
|--------|------|
| SKILL.md 格式 | 符合 agentskills.io 规范 |
| Frontmatter | 包含 name 和 description |
| 目录结构 | 符合标准（references/scripts/assets） |
| 依赖项 | 无特殊依赖或依赖可安装 |

### 4. 可改编性

| 因素 | 易改编 | 难改编 |
|------|--------|--------|
| 代码复杂度 | 简单脚本 | 复杂系统 |
| 领域相关性 | 同领域 | 跨领域 |
| 模块化程度 | 高 | 低 |
| 文档完整度 | 完整 | 不完整 |

## 决策流程

```
搜索结果
    ↓
有匹配结果？
    ├─ 否 → 从零创建
    └─ 是 → 评估匹配度
        ↓
        匹配度 > 80%？
        ├─ 是 → 质量指标良好？
        │   ├─ 是 → 【推荐安装】直接安装现有 Skill
        │   └─ 否 → 【推荐改编】基于现有 Skill 改编
        └─ 否 → 匹配度 50-80%？
            ├─ 是 → 【推荐改编】基于现有 Skill 改编
            └─ 否 → 【推荐参考】参考设计，从零创建
```

## 决策说明

### 推荐安装

**条件**: 匹配度 > 80% 且质量指标良好

**操作**:
1. 向用户展示 Skill 详情
2. 确认安装位置
3. Clone 或复制到目标目录
4. 验证安装成功

**示例输出**:
```
找到匹配的 Skill:
- 名称: pdf-toolkit
- 描述: PDF 处理工具集（提取、合并、转换）
- 来源: https://github.com/xxx/pdf-toolkit
- Star: 156
- 更新: 2 周前
- 匹配度: 92%

建议: 直接安装此 Skill，无需修改。
是否安装？
```

### 推荐改编

**条件**: 匹配度 50-80% 或质量指标一般

**操作**:
1. 向用户展示 Skill 详情
2. 说明需要修改的部分
3. 确认改编方案
4. Clone 并修改
5. 注明来源

**示例输出**:
```
找到可改编的 Skill:
- 名称: code-reviewer
- 描述: 代码审查助手
- 来源: https://github.com/xxx/code-reviewer
- Star: 89
- 更新: 1 月前
- 匹配度: 65%

需要修改:
1. 添加安全审查规则
2. 更新检查清单
3. 适配项目规范

建议: 基于此 Skill 改编，节省 60% 工作量。
是否改编？
```

### 推荐参考

**条件**: 匹配度 < 50%

**操作**:
1. 向用户展示参考 Skill
2. 说明可借鉴的部分
3. 从零创建，参考设计

**示例输出**:
```
找到可参考的 Skill:
- 名称: data-analyzer
- 描述: 数据分析工具
- 来源: https://github.com/xxx/data-analyzer
- 匹配度: 35%

可借鉴:
1. 目录结构设计
2. Quick Reference 格式
3. 示例写法

建议: 参考设计，从零创建。
是否继续？
```
