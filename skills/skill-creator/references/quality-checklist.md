# Skill 质量检查清单

## 快速检查（2 分钟）

### Frontmatter
- [ ] 有 YAML frontmatter
- [ ] `name` 匹配 `^[a-z][a-z0-9-]*$`
- [ ] `name` 与目录名一致
- [ ] `description` 包含"做什么 + 何时用"

### 必须章节
- [ ] 有 "When to Use This Skill"
- [ ] 有 "Not For / Boundaries"
- [ ] 有 "Quick Reference"
- [ ] 有 "Examples"（>= 3 个）

## 详细检查（10 分钟）

### 激活可靠性
- [ ] 触发条件是具体的、可判定的
- [ ] 包含具体关键词（不只是"当需要时"）
- [ ] 边界清晰，不会误触发

### Quick Reference 质量
- [ ] <= 20 个模式
- [ ] 每个模式可以直接复制使用
- [ ] 没有大段解释文字（应该在 references/）

### Examples 质量
- [ ] >= 3 个端到端示例
- [ ] 每个示例有输入、步骤、验收标准
- [ ] 示例可以复现

### 内容组织
- [ ] 长内容在 `references/`
- [ ] 有 `references/index.md` 导航
- [ ] 读起来像操作手册

### 诚实性
- [ ] 没有凭空发明的内容
- [ ] 不确定的声明有验证路径
- [ ] 标明来源和最后更新日期

## 评分标准

| 分数 | 评价 |
|:---|:---|
| 9-10 | 优秀，可以直接发布 |
| 7-8 | 良好，小修改后可发布 |
| 5-6 | 及格，需要较大改进 |
| < 5 | 不合格，需要重写 |

## 常见问题修复

### 问题：触发不可靠

**症状**: Skill 有时触发有时不触发

**修复**:
1. 检查 `description` 是否包含具体关键词
2. 在 "When to Use" 中添加更多触发条件
3. 确保关键词足够独特

### 问题：Quick Reference 太长

**症状**: 超过 20 个模式或包含大段文字

**修复**:
1. 只保留最常用的 10-20 个模式
2. 大段解释移到 `references/`
3. 每个模式精简到可复制使用

### 问题：示例不可复现

**症状**: 按照示例操作无法得到预期结果

**修复**:
1. 检查示例的前提条件是否完整
2. 添加环境/依赖说明
3. 实际运行一遍验证
