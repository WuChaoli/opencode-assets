# Agent 重构模式

## 模式 1: 权限优化

**症状**: Agent 有不必要的权限或缺少必要权限

**修复**:
1. 检查当前 tools/permission 配置
2. 按最小权限原则调整
3. 验证修改后行为

**示例**:
```yaml
# Before
permission:
  edit: allow
  bash: allow

# After
permission:
  edit: deny
  bash: ask
```

## 模式 2: Prompt 精简

**症状**: Prompt 超过 100 行，角色模糊

**修复**:
1. 重写 Role 为 1-2 句话
2. 精简 Responsibilities 到 <= 5 条
3. 移除冗余说明

## 模式 3: 资源更新

**症状**: Available Resources 过时或不相关

**修复**:
1. 检查 Skills/MCP/Tools 是否仍适用
2. 移除不相关资源
3. 添加新相关资源

## 模式 4: 触发优化

**症状**: Agent 有时触发有时不触发

**修复**:
1. 检查 description 是否包含具体关键词
2. 添加更多触发场景描述
3. 确保关键词足够独特

## 模式 5: 参数调优

**症状**: temperature/steps/model 不匹配任务

**修复**:
1. 分析任务类型
2. 调整参数到合适值
3. 验证修改后行为

## 模式 6: 职责聚焦

**症状**: Agent 做了太多事

**修复**:
1. 识别核心职责
2. 移除不相关职责
3. 如需要，拆分为多个 Agent

## 模式选择决策树

```
权限配置是否合理？
  ├─ 否 → 模式 1: 权限优化
  └─ 是 → Prompt 是否 > 100 行？
      ├─ 是 → 模式 2: Prompt 精简
      └─ 否 → 资源是否过时？
          ├─ 是 → 模式 3: 资源更新
          └─ 否 → 触发是否可靠？
              ├─ 否 → 模式 4: 触发优化
              └─ 是 → 参数是否合适？
                  ├─ 否 → 模式 5: 参数调优
                  └─ 是 → 职责是否聚焦？
                      ├─ 否 → 模式 6: 职责聚焦
                      └─ 否 → 可能不需要修改
```
