# Agent 评估标准与决策流程

## 评估维度

### 1. 功能匹配度（0-100%）

| 分数 | 标准 |
|------|------|
| 90-100% | 完全覆盖用户需求，几乎不需要修改 |
| 70-89% | 覆盖主要需求，需要少量修改 |
| 50-69% | 覆盖部分需求，需要较多修改 |
| 30-49% | 只有部分相关，需要大量修改 |
| 0-29% | 基本不相关 |

### 2. 质量指标

| 指标 | 优秀 | 良好 | 一般 | 较差 |
|------|------|------|------|------|
| 下载量 | > 5000 | 1000-5000 | 100-1000 | < 100 |
| 评分 | > 4.5 | 4.0-4.5 | 3.5-4.0 | < 3.5 |
| 更新日期 | < 1 月 | 1-3 月 | 3-6 月 | > 6 月 |
| 文档质量 | 完整清晰 | 基本完整 | 不完整 | 缺失 |

### 3. 兼容性

| 检查项 | 标准 |
|--------|------|
| 格式 | 符合 OpenCode agent markdown 格式 |
| Frontmatter | 包含 description 和 mode |
| 章节 | 包含 Role/Responsibilities/Constraints |
| 依赖项 | 无特殊依赖或依赖可安装 |

### 4. 可改编性

| 因素 | 易改编 | 难改编 |
|------|--------|--------|
| Prompt 复杂度 | 简单清晰 | 复杂冗长 |
| 领域相关性 | 同领域 | 跨领域 |
| 模块化程度 | 高 | 低 |
| 文档完整度 | 完整 | 不完整 |

## 决策流程

```
搜索结果
    ↓
有匹配结果？
    ├─ 否 → 从零创建
    └─ 是 → 评估匹配度
        ↓
        匹配度 > 80%？
        ├─ 是 → 质量指标良好？
        │   ├─ 是 → 【推荐安装】直接安装现有 Agent
        │   └─ 否 → 【推荐改编】基于现有 Agent 改编
        └─ 否 → 匹配度 50-80%？
            ├─ 是 → 【推荐改编】基于现有 Agent 改编
            └─ 否 → 【推荐参考】参考设计，从零创建
```

## 决策说明

### 推荐安装

**条件**: 匹配度 > 80% 且质量指标良好

**操作**:
1. 向用户展示 Agent 详情
2. 确认安装位置
3. 下载或复制到目标目录
4. 验证安装成功

### 推荐改编

**条件**: 匹配度 50-80% 或质量指标一般

**操作**:
1. 向用户展示 Agent 详情
2. 说明需要修改的部分
3. 确认改编方案
4. 下载并修改
5. 注明来源

### 推荐参考

**条件**: 匹配度 < 50%

**操作**:
1. 向用户展示参考 Agent
2. 说明可借鉴的部分
3. 从零创建，参考设计
