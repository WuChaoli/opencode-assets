# Agent 质量检查清单

## 快速检查（2 分钟）

### Frontmatter
- [ ] 有 YAML frontmatter
- [ ] `description` 包含"做什么 + 何时用"
- [ ] `mode` 已指定（primary/subagent/all）

### 必须章节
- [ ] 有 "## Role"
- [ ] 有 "## Responsibilities"
- [ ] 有 "## Available Resources"
- [ ] 有 "## Constraints"

### 资源推荐
- [ ] Available Resources 包含 Skills/MCP/Tools 至少一项
- [ ] 资源推荐聚焦，不超过 5 项

## 详细检查（10 分钟）

### 权限配置
- [ ] 权限最小化（只给必要的）
- [ ] subagent 默认 edit: ask, bash: ask
- [ ] 只读 agent 设为 edit: deny, bash: deny

### 描述质量
- [ ] 描述包含具体触发场景
- [ ] 描述长度 > 10 字符
- [ ] 不使用模糊描述

### 职责聚焦
- [ ] 职责不超过 5 条
- [ ] 每条职责具体可执行
- [ ] 没有重叠或模糊的职责

### 约束具体化
- [ ] 约束具体可执行
- [ ] 包含安全约束（如不修改关键文件）
- [ ] 不包含空泛的"小心一点"

### 资源推荐质量
- [ ] 只推荐 agent 会用到的资源
- [ ] 每个资源有用途说明
- [ ] 没有罗列所有可用资源

## 评分标准

| 分数 | 评价 |
|:---|:---|
| 9-10 | 优秀，可以直接发布 |
| 7-8 | 良好，小修改后可发布 |
| 5-6 | 及格，需要较大改进 |
| < 5 | 不合格，需要重写 |

## 常见问题修复

### 问题：agent 触发不可靠

**症状**: agent 有时触发有时不触发

**修复**:
1. 检查 `description` 是否包含具体场景
2. 在 description 中添加更多触发关键词

### 问题：权限过度

**症状**: agent 有不需要的写权限或 bash 权限

**修复**:
1. 检查 permission 配置
2. 设为最小必要权限

### 问题：资源罗列

**症状**: Available Resources 列出大量不相关资源

**修复**:
1. 只保留 agent 会用到的 2-5 个资源
2. 每个资源必须有用途说明
