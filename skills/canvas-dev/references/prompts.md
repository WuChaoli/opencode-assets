# Canvas 驱动开发提示词集合

## 架构分析提示词

### 01-快速架构分析

```markdown
分析当前项目的架构，生成 Mermaid 架构图。

要求：
- 粒度：服务级（聚合相关文件）
- 关注：模块边界、依赖关系、数据流
- 颜色编码：数据库=绿色，缓存=红色，外部服务=蓝色
- 输出洞察摘要：架构模式、核心组件、潜在风险
```

### 02-详细架构分析

```markdown
作为 AI 架构总师，对当前项目进行深度架构分析。

执行五阶段流程：
1. 全局感知：扫描所有源文件，构建依赖网络
2. 粒度决策：根据项目规模选择合适抽象级别
3. 语义分析：识别组件角色（入口/服务/数据访问）
4. 布局规划：按层级组织（前端→API→服务→数据）
5. 质量优化：减少交叉，突出关键路径

输出：
- Canvas JSON 格式白板文件
- 洞察摘要报告
```

### 03-遗留项目分析

```markdown
这是一个遗留项目，我需要快速理解它的架构。

请：
1. 扫描项目结构，识别技术栈
2. 分析核心业务流程
3. 识别技术债务和风险点
4. 生成架构白板，标注：
   - 核心模块（高亮）
   - 高风险区域（红色警告）
   - 建议的切入点

输出粒度：文件级
```

## 白板驱动编码提示词

### 04-根据白板生成代码

```markdown
我在白板上画了以下架构：

[粘贴 Canvas JSON 或描述新增模块]

请根据白板实现代码：
1. 创建节点对应的文件/类
2. 实现连线表示的调用关系
3. 遵循项目现有代码风格
4. 生成必要的类型定义和接口

注意：不要修改白板中未涉及的现有代码。
```

### 05-白板驱动重构

```markdown
我在白板上修改了架构：

[描述修改内容]
- 拆分：{原模块} → {新模块A} + {新模块B}
- 新增连线：{A} → {B}
- 删除连线：{X} → {Y}

请按新架构重构代码：
1. 列出需要修改的文件清单
2. 逐个文件给出修改方案
3. 保持现有功能不变
4. 更新相关的 import 语句
```

### 06-新模块设计

```markdown
我需要添加一个新模块：{模块名}

白板位置：
- 调用者：{上游模块}
- 被调用：{下游模块}
- 数据源：{数据库/缓存/外部API}

职责描述：
{一句话描述模块职责}

请：
1. 设计模块接口（输入/输出）
2. 生成代码框架
3. 更新白板 JSON（新增节点和连线）
```

## 白板同步检查提示词

### 07-一致性检查

```markdown
检查当前代码与白板的一致性。

白板文件：{path/to/architecture.canvas}

检查项：
1. 白板中的节点是否都有对应代码文件？
2. 白板中的连线是否反映真实的 import 关系？
3. 是否有代码文件未在白板中体现？
4. 是否有实际依赖未在白板中标注？

输出格式：
- ✅ 一致项
- ⚠️ 不一致项（附修复建议）
- 📝 建议更新
```

### 08-白板更新建议

```markdown
最近我修改了以下代码：

[列出修改的文件/功能]

请分析这些修改对架构的影响，建议白板更新：
1. 需要新增的节点
2. 需要新增/删除的连线
3. 需要更新的节点描述
4. 颜色编码调整建议
```

## Code Review 提示词

### 09-架构视角 Review

```markdown
从架构角度 Review 这个 PR/变更：

变更内容：
[描述或粘贴 diff]

参考当前架构白板，检查：
1. 是否引入了不合理的依赖？
2. 是否破坏了模块边界？
3. 是否存在循环依赖风险？
4. 是否符合现有架构模式？

输出：
- 架构影响评估
- 风险点标注
- 改进建议
```

### 10-依赖关系审查

```markdown
审查这个模块的依赖关系：

模块：{module_name}
文件：{file_path}

检查：
1. 依赖方向是否正确？（不应该依赖上层）
2. 是否有不必要的依赖？
3. 是否应该抽象接口而非直接依赖实现？
4. 在白板中标注可疑的依赖关系
```

## 团队协作提示词

### 11-架构讲解

```markdown
基于当前架构白板，为新人生成架构讲解文档：

目标读者：{新入职开发者/外部合作方/产品经理}

内容要求：
1. 系统整体介绍（一段话）
2. 核心模块说明（每个模块 2-3 句）
3. 关键数据流解释
4. 常见问题解答

语言风格：简洁明了，避免过度技术术语
```

### 12-技术方案模板

```markdown
基于当前架构，为以下需求设计技术方案：

需求描述：
{需求内容}

请输出：
1. 在白板上标注影响范围
2. 新增/修改的模块设计
3. 接口定义
4. 数据流变化
5. 风险评估
6. 工作量估算
```

## 快捷指令

| 指令 | 用途 |
|:---|:---|
| `/arch` | 快速生成 Mermaid 架构图 |
| `/canvas` | 生成 Canvas JSON 白板 |
| `/sync` | 检查代码-白板一致性 |
| `/expand {模块}` | 展开模块内部细节 |
| `/risks` | 标注架构风险点 |
| `/explain` | 生成架构讲解文档 |
